# promobot

Модуль отвечает за выдачу промокодов в чатах по ключевым словам.

## Что делает
- Проверяет входящие сообщения и ищет ключевые слова по подстроке.
- При совпадении отправляет заранее заданный текст ответа.
- Работает в Telegram и MAX (отдельные боты).
- Привязка чатов через команду `/bind CODE`.

## Точки входа
- UI: `/adm/index.php?m=promobot`
- TG webhook: `/adm/modules/promobot/webhook.php?bot_id=<id>`
- MAX webhook: `/adm/modules/promobot/max_webhook.php?bot_id=<id>`

## Установка
1. Применить SQL: `adm/modules/promobot/install.sql`.
2. Открыть `/adm/index.php?m=promobot`.
3. Создать бота (Telegram или MAX).
4. Заполнить токены/ключи в настройках бота.
5. Для Telegram нажать «Подключить webhook».
6. Для MAX настроить webhook на URL модуля и подписаться на события сообщений.
7. Сгенерировать код привязки и отправить `/bind CODE` в нужный чат.
8. Добавить промокоды (ключевые слова через запятую + текст ответа).

Для уже установленного модуля (обновление схемы):
1. Применить SQL: `adm/modules/promobot/update.sql`.
2. В настройках нужного бота выбрать «Источник списка промокодов».

## Общий список промокодов
- У каждого бота можно выбрать источник промокодов: свой список или другой бот.
- Если выбран другой бот, промокоды в выбранном боте и в webhook берутся из источника.
- Это позволяет использовать один список одновременно в Telegram и MAX без дублирования.

## Роли
- admin/manager: полный доступ (боты, привязки, пользователи, промокоды).
- user: доступ только к назначенным ботам и их промокодам.

## Логирование
- Включается/выключается кнопкой на странице модуля.
- Логи webhook пишутся в таблицу `promobot_logs` (если включено).
- Фиксируются все входящие сообщения, включая случаи без совпадений.

## Важно
- Если чат не привязан — ответы не отправляются.
- При отсутствии совпадений бот молчит.
