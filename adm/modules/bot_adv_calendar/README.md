# bot_adv_calendar

Telegram-бот рекламного календаря.

## Привязка ролей
- `user` (пользователь CRM-настройщик): двухэтапно.
  1) Сначала подключение пользователя к модулю в CRM (таблица `bot_adv_calendar_user_access`).
  2) Затем, при необходимости, выдача 6-значного TG-кода `/start <code>`.
- `client` (покупатель рекламы): без кода, через первичный онбординг в боте.

## Клиентский поток (новый)
1. Клиент пишет боту впервые.
2. Бот запрашивает имя.
3. Бот запрашивает телефон.
4. Система ищет клиента по телефону.
5. Если найден — привязывает `chat_id` к существующему `clients.id`.
6. Если не найден — автоматически создает клиента в `clients` и сразу привязывает.
7. После привязки клиент получает меню.
8. Кнопка `Календарь рекламы` показывает Telegram-календарь по месяцам с цветами доступности дней:
   - `🟢` свободно
   - `🟡` мало мест
   - `🔴` нет мест

## Что есть в админке
- Настройки бота (`enabled`, `bot_token`, `webhook_secret`, `webhook_url`, TTL, retention).
- Установка/проверка webhook.
- Подключение пользователей CRM к модулю и их отключение.
- Генерация TG-кода только для уже подключенных пользователей CRM.
- Таблицы подключенных пользователей и привязанных клиентов.

## CRM-режим для пользователя
- Роль `user` может работать в модуле без Telegram.
- После подключения пользователя к модулю доступны:
  - режим бронирования (`fixed` / `range`);
  - интервал слотов;
  - рабочее время и цикличные выходные;
  - тарифные окна (день недели, время, цена).
- Telegram-привязка для `user` остается опциональной.

## SQL установка
Выполнить вручную:
- `adm/modules/bot_adv_calendar/install.sql`
- `adm/modules/bot_adv_calendar/update.sql` (если модуль уже был установлен ранее)

## Точки входа
- UI: `/adm/index.php?m=bot_adv_calendar`
- webhook: `/adm/modules/bot_adv_calendar/webhook.php`

## Важно
- Модуль не вводит новые статусы заявок.
- Логика календаря/записей должна использовать существующие API `requests`, `calendar`, `services`, `clients`.
