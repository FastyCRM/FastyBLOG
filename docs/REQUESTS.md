# REQUESTS (модуль заявок)

## 1) Назначение
- Канбан‑доска заявок (статусы: new → confirmed → in_work → done).
- Архивирование (без удаления).
- Комментарии и история.
- Интеграция с услугами/специалистами.
- Внешняя заявка (site) через internal API.

## 2) Роли и доступ
- **admin/manager**: видят все, подтверждают, берут в работу, закрывают.
- **specialist/user**: видят только свои заявки (назначенные/взятые).

## 3) Настройки модуля (requests_settings)
- `use_specialists` (0/1) — включение режима специалистов.
- `use_time_slots` (0/1) — интервальный режим (слоты времени).

Режимы:
- Если `use_specialists=0` — заявка без выбора специалиста.
- Если `use_specialists=1` — обязательны `service_id` и `specialist_user_id`.
- Если `use_time_slots=1` — время выбирается из слотов календаря.

## 4) Основные потоки
### Создание заявки (CRM)
- Вводится имя/телефон, опционально email.
- Если клиента нет — создаётся в `clients`.
- При наличии `service_id`/`specialist_user_id` проверяется связь в `user_services`.

### Подтверждение
- Перевод в `confirmed`.
- Фиксация даты/времени/стоимости.

### Взять в работу
- Перевод в `in_work`.
- Если заявка назначена специалисту — взять может только он.

### Закрытие (done)
- Перевод в `done`.
- Создаётся счёт (опционально) + позиции.
- Автонумерация: `INV-<id>-YYYYMMDD`, `ACT-<id>-YYYYMMDD`, `CTR-<id>-YYYYMMDD`.
- Разрешена «свободная» услуга: создаётся в `services` и попадает в «Без категории».

### Архив
- Без удаления, только `archived_at` и `archived_reason`.

## 5) Внешняя форма (landing)
Используется internal API:
- `GET /core/internal_api.php?m=requests&do=api_services`
- `POST /core/internal_api.php?m=requests&do=api_request_add`

Обязательное согласие на обработку данных (`consent=1`).

## 6) Таблицы
- `requests`
- `request_history`
- `request_comments`
- `request_invoices`
- `request_invoice_items`

Ключевые поля:
- `requests.slot_key` — защита от дублей (уникальный ключ слота).
- `requests.visit_at` — дата/время визита.

## 7) UI
- Канбан‑доска с 4 колонками.
- Модалки: создание, просмотр, настройки.
- Все настройки модуля через шестерню (modal_settings).
